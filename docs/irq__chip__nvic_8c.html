Detailed Description

NVIC-backed irq_chip implementation for ARM Cortex-M (STM32)

Author
Subhajit Roy (subha.nosp@m.jitr.nosp@m.oy005.nosp@m.@gma.nosp@m.il.co.nosp@m.m)

This module implements the generic IRQ controller abstraction (struct irq_chip) using the ARM Cortex-M NVIC hardware.

It provides a translation layer between:

  • Generic IRQ subsystem (irq_desc / irq_data)
  • STM32 HAL NVIC APIs

This allows upper layers to remain platform-independent while delegating interrupt control to the NVIC.

IRQ Execution Flow

Typical interrupt handling path:

Vector Table → ISR (startup.s)
→ HAL IRQ Handler
→ Generic IRQ Dispatcher
→ irq_chip callbacks (this file)

NVIC Hardware Constraints

  • No explicit ACK mechanism
  • No End-Of-Interrupt (EOI) signaling required
  • Interrupt trigger type is fixed (peripheral-defined)
  • Masking == disabling at NVIC level
  • Priority-based preemption supported
Note
Compiled only when CONFIG_DEVICE_VARIANT == MCU_VAR_STM
