Detailed Description

STM32 RCC + CMSIS system initialization backend.

Author
Subhajit Roy (subha.nosp@m.jitr.nosp@m.oy005.nosp@m.@gma.nosp@m.il.co.nosp@m.m)

This module implements the Reset and Clock Control (RCC) backend for STM32, replacing the default CMSIS system file (e.g. system_stm32f4xx.c).

It provides a clean separation between:

  • Early boot clock setup (pre-scheduler)
  • Runtime clock configuration (post HAL_Init)

The implementation integrates:

  • CMSIS SystemCoreClock model
  • STM32 HAL RCC APIs
  • Project-specific driver abstraction (drv_rcc)

Pre-Scheduler Initialization

Executed before C runtime initialization (from Reset_Handler):

  • SystemInit()
    • Enables FPU (if present)
    • Optionally relocates vector table (VTOR)
  • SystemCoreClockUpdate()
    • Reads RCC registers
    • Updates global SystemCoreClock
  • CMSIS globals (required by HAL):
    • SystemCoreClock
    • AHBPrescTable[]
    • APBPrescTable[]

Post-HAL Initialization

Executed after HAL_Init() from application layer:

  • _stm32_clock_init() Configure PLL, bus clocks, and flash latency
  • Clock query APIs: _stm32_get_sysclk_hz() _stm32_get_apb1_hz() _stm32_get_apb2_hz()

Configured Clock Tree

Default configuration:

HSI (16 MHz)
└─ /PLLM → VCO input
└─ ×PLLN → VCO output
├─ /PLLP → SYSCLK
└─ /PLLQ → USB/SDIO
AHB = SYSCLK / 1
APB1 = HCLK / 2
APB2 = HCLK / 1

Design Constraints

  • All RCC register access is encapsulated in this file
  • Upper layers must not directly use HAL_RCC_* APIs
  • Peripheral clock enables are centralized here
Note
Compiled only when CONFIG_DEVICE_VARIANT == MCU_VAR_STM

Update SystemCoreClock from RCC registers.

Computes the current system clock frequency based on RCC configuration. This function is required by CMSIS and is used by STM32 HAL internally.

Note
Must be called whenever clock configuration changes.
Warning
Assumes PLL and prescaler configuration follow STM32 reference manual.
