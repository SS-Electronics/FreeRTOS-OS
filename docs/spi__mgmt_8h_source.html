1/*
2 * spi_mgmt.h — SPI management service thread
3 *
4 * This file is part of FreeRTOS-OS Project.
5 *
6 * The SPI management thread serialises all SPI bus transactions. Because
7 * SPI is full-duplex the message carries both a TX and RX buffer; callers
8 * may set either to NULL for transmit-only or receive-only operations.
9 *
10 * Chip-select (CS) control is the caller's responsibility — the SPI
11 * management layer does not manage CS GPIO pins. Assert CS before posting
12 * a message and deassert it after receiving the completion notification.
13 */
14
