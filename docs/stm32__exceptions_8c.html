Detailed Description

Cortex-M4 core exception handlers for STM32.

Author
Subhajit Roy (subha.nosp@m.jitr.nosp@m.oy005.nosp@m.@gma.nosp@m.il.co.nosp@m.m)

Strong definitions for Cortex-M core exception handlers that override the weak symbols in startup_stm32f411vetx.c.

Fault handler chain (HardFault / MemManage / BusFault / UsageFault):

CPU fires exception │ ▼ naked trampoline (_Handler) determine active SP from EXC_RETURN bit[2]: bit[2]==0 → MSP was active (running in handler or MSP task) bit[2]==1 → PSP was active (running in a FreeRTOS task) │ ▼ _fault_handler_c(frame*, exc_return, fault_type) capture hardware exception frame (r0–r3, r12, lr, pc, xpsr) capture SCB->HFSR, CFSR, BFAR, MMFAR write g_fault_info (visible in debugger / crash dump) emit diagnostic text directly via USART2 (no ring buffer, no RTOS) halt in spin loop

Note
This module is compiled only when CONFIG_DEVICE_VARIANT == MCU_VAR_STM.
